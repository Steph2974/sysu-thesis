\section{研究背景与意义}
\label{sec:background}

随着移动互联网、智能终端与定位技术的快速发展，基于位置的服务（Location-Based Services, LBS）已经成为人们获取空间信息、发现周边资源和进行城市活动决策的重要入口。地图搜索、导航推荐、本地生活服务、社交签到和旅游出行等应用场景每天都会产生大量带有时空属性的用户行为数据。与传统互联网推荐任务相比，位置相关行为天然包含用户兴趣、时间上下文以及地理空间约束等多重信息，因此如何从这类复杂行为数据中刻画用户移动规律、理解用户即时需求，并进一步实现高质量的位置推荐，已成为推荐系统、数据挖掘和城市计算领域的重要研究课题。

基于位置的社交网络（Location-Based Social Networks, LBSNs）为这一研究提供了典型的数据基础。LBSNs 通常记录用户在真实地理空间中的签到、搜索、点击与访问行为，使得研究者能够从用户轨迹中分析其兴趣偏好、活动区域以及时空迁移模式。围绕 LBSNs 展开的点兴趣（Point-of-Interest, POI）推荐研究，不仅服务于个性化推荐系统本身，也对智慧城市、商业选址、区域活力分析、交通调度和公共服务优化等任务具有现实意义。已有研究表明，LBSNs 与 POI 推荐问题具有信息源异构、数据稀疏、空间依赖显著和上下文敏感性强等特点，因此不能简单地将其视作传统物品推荐的直接延伸，而需要结合时空特征进行针对性建模 \cite{zhao2016survey,sanchez2021survey}。

% 在 POI 推荐任务中，下一个兴趣点预测（Next POI Recommendation）是最具代表性的研究问题之一。该任务旨在根据用户历史访问序列和当前上下文，预测其下一步最可能访问的兴趣点。与一般序列推荐不同，Next POI 推荐不仅要建模用户历史兴趣和行为顺序，还必须考虑 POI 之间的地理距离、区域可达性以及时间规律等因素。现实场景中，用户下一步访问行为往往受到短时活动半径和空间连续性的显著影响，因此单纯依赖语义相似性或协同过滤信号通常难以准确刻画真实移动模式。尤其在城市环境中，不同区域的同类 POI 虽然在类别语义上相近，但其对用户的实际吸引力和可达性可能存在明显差异，这使得地理信息成为 POI 推荐不可忽视的核心因素。

近年来，针对 Next POI 推荐问题，研究者从序列建模、图结构建模、时空联合建模和大模型建模等多个角度进行了探索。早期方法主要依赖马尔可夫链、矩阵分解或基于距离衰减的排序机制，后来逐步发展到循环神经网络、自注意力网络、图神经网络以及超图学习等方法，以提升对用户复杂时空行为的表达能力 \cite{feng2018deepmove,kang2018sasrec,luo2021stan,wang2022graph,yan2023sthgcn}。这些方法在建模用户历史迁移规律方面取得了显著进展，但在 POI 表示学习层面仍然存在明显局限：大量工作仍将 POI 视作原子级离散 ID，或者仅使用类别标签、区域标签等有限侧信息进行编码，导致模型难以充分表达 POI 的深层语义关联及其潜在空间结构。

随着大语言模型（Large Language Models, LLMs）与生成式检索（Generative Retrieval, GR）的兴起，推荐系统研究正逐渐从判别式排序范式转向生成式建模范式。生成式推荐将目标物品预测重构为一个序列生成问题，使模型能够在统一 token 空间中处理用户行为、上下文条件与目标对象之间的关系。相比传统的召回---排序流水线，生成式推荐在统一建模复杂上下文和捕获隐式意图方面展现出较大潜力 \cite{wang2023genrecsurvey,rajput2023tiger,li2023llmgenrecsurvey}。在这一背景下，语义标识（Semantic ID, SID）逐渐成为生成式推荐中的关键概念。与随机数字 ID 不同，SID 试图为物品构造具有结构性和语义性的离散标识，使相似对象能够在离散 token 空间中共享更长的前缀，从而帮助生成模型学习更稳定的上下文---目标映射关系 \cite{rajput2023tiger}。

% 在 POI 场景中，GNPR-SID 将 Semantic ID 的思想引入 Next POI 推荐任务，提出通过 RQ-VAE 将 POI 的多源特征量化为多层级离散语义标识，再利用大语言模型根据历史访问记录生成下一兴趣点的 SID。该方法为生成式 POI 推荐提供了新的技术路径，也说明了在 POI 任务中构建结构化语义 ID 的可行性 \cite{wang2025gnprsid}。然而，现有基于 SID 的方法仍然面临一个重要问题：POI 本质上同时具有语义属性和地理属性，但现有离散表示学习往往更加关注类别、区域和协同信号，而缺乏对真实地理空间关系的充分建模。若地理信息仅以粗粒度区域标签或简单附加特征的形式参与建模，则最终学得的 SID 更偏向于“语义离散表示”，而难以充分体现 POI 的空间差异和邻近关系。

% 从应用需求出发，这一问题尤其值得关注。地图搜索和位置推荐场景中的用户查询往往具有较强的空间敏感性和情境依赖性。用户所关注的不只是“该地点属于什么类别”，还包括距离、活动区域与行程约束等因素。因而，在 Next POI 推荐中，兴趣点的离散目标表示若仅反映类别或协同语义，而难以体现真实空间结构与邻近关系，将限制模型对用户空间决策过程的刻画能力。

围绕生成式推荐与语义 ID 的研究表明，目标对象的离散编码方式会显著影响大模型的学习与泛化；在 POI 场景下，地理因素如何进入\emph{语义 ID 的形成过程}，而不仅停留在预测或排序阶段的外部特征层面，仍是一个有待深入讨论的问题。本文在该问题设定下，以 GNPR-SID 所代表的生成式语义 ID 路线为参照，将研究重心置于语义 ID 构造环节，探索增强 POI 离散表示地理感知能力的路径，并为下一兴趣点预测提供更具空间一致性的目标表示。具体模型结构、地理信息引入方式及与基线框架的对照，均在第三章展开。

\section{国内外研究现状和相关工作}
\label{sec:related_work}

\subsection{POI 推荐与 Next POI 任务}

基于位置的社交网络（Location-Based Social Networks, LBSNs）为兴趣点推荐研究提供了典型的数据载体。与传统物品推荐不同，POI 推荐中的用户行为同时包含用户兴趣偏好、访问时间、空间位置以及轨迹转移规律，因此该任务兼具推荐系统、时空数据挖掘与城市计算等多重属性。Zhao 等较早系统总结了 LBSNs 场景下 POI 推荐的核心问题，指出该领域长期受到数据稀疏、冷启动、用户偏好动态变化和显著地理约束等因素的共同影响 \cite{zhao2016survey}。S{\'a}nchez 等进一步从实验视角综述了 POI 推荐的研究范式与评测方法，指出 POI 推荐不能简单视作传统推荐任务在位置场景下的直接迁移，而必须显式建模时间因素与空间因素 \cite{sanchez2021survey}。随着移动终端和定位服务的普及，POI 推荐已从早期的静态偏好预测逐渐发展为面向动态用户行为的时空决策问题。

在各类 POI 推荐任务中，下一个兴趣点预测（Next Point-of-Interest Recommendation）是最具代表性的研究问题之一。该任务通常根据用户历史签到序列、当前时刻及上下文信息，预测用户下一步最可能访问的地点。相比一般序列推荐，Next POI 推荐不仅要刻画用户行为顺序，还必须关注地点之间的空间可达性、活动半径和区域迁移模式，因此天然具有更强的时空约束。早期研究多依赖马尔可夫链、矩阵分解或基于距离衰减的排序机制，虽然具有一定可解释性，但难以表达复杂的长期依赖与高阶空间关系。近年来，研究者逐步采用更强的深度模型对用户移动过程进行建模。Li 等提出 Seq2Graph Augmentation，通过在轨迹序列之间传播协同信号来缓解 POI 转移稀疏问题 \cite{li2021seq2graph}；Wang 等提出图增强时空网络，从图结构视角显式建模地理依赖和顺序依赖 \cite{wang2022graph}；Lim 等进一步引入层次化多任务学习思想，在不同空间粒度上学习用户区域偏好，以缩小搜索空间并提高下一 POI 预测效率 \cite{lim2022hmtgrn}。此外，Wang 等的解耦图表示方法、Yan 等的时空超图建模方法以及 Lai 等的解耦对比超图学习方法，分别从影响因素解耦、高阶关系刻画和多方面协同建模等角度推动了 Next POI 推荐的发展 \cite{wang2022dran,yan2023sthgcn,lai2024dchl}。
总体来看，Next POI 推荐研究已经从基于简单转移概率的浅层建模，逐渐发展为融合协同信号、图结构、高阶关系和复杂上下文的深度建模范式。然而，这类方法的关注重点大多集中在“如何更好地表示用户访问序列”以及“如何更精细地利用时空关系”，而对 POI 本身的离散表示方式关注相对不足。大量工作仍将 POI 作为原子级 ID 处理，或者仅使用类别标签、区域标签等有限侧信息进行辅助建模，这在一定程度上限制了模型进一步理解 POI 深层语义结构与空间结构的能力。

\subsection{序列推荐与 POI 序列建模}

从更一般的推荐系统研究视角来看，Next POI 推荐本质上属于序列推荐问题在位置服务场景下的特化形式。序列推荐的核心目标是根据用户历史交互序列预测未来偏好，关键在于刻画兴趣随时间的演化过程及不同行为之间的依赖关系。Kang 等提出的 SASRec 证明了自注意力机制在序列推荐中的有效性，相比传统循环神经网络能够更灵活地捕捉长距离依赖 \cite{kang2018sasrec}。此后，围绕序列推荐的研究在表示学习、自监督增强、多行为建模和鲁棒训练等方面持续深入。Xie 等利用对抗学习与对比学习增强变分自编码器在序列推荐中的表示能力 \cite{xie2021acvae}；Liu 等提出 CoSeRec，通过鲁棒数据增强与对比自监督学习缓解序列稀疏与噪声问题 \cite{liu2021coserec}；Zhou 等则提出 FMLP-Rec，表明即使不依赖复杂的自注意力结构，也可以通过合适的滤波与前馈设计获得具有竞争力的序列建模效果 \cite{zhou2022fmlprec}。这些工作为理解用户行为演化、处理长序列依赖和缓解数据稀疏提供了重要方法基础。

在 POI 场景中，由于用户行为序列同时具有时间间隔、空间间隔和多粒度上下文，序列建模通常比一般商品推荐更复杂。已有研究围绕这一特点提出了多种针对性结构。Luo 等提出 STAN，在自注意力框架中显式融入时间间隔和空间距离，使模型在分配注意力权重时能够直接感知时空差异 \cite{luo2021stan}。Wang 等提出 STiSAN，通过轻量机制联合建模空间间隔与时间间隔，提升模型在时空异质场景中的适应能力 \cite{wang2022stisan}。Xie 等使用层次化 Transformer 聚合不同尺度的时空上下文，以增强对短期行为模式与局部结构特征的建模能力 \cite{xie2022htstca}。除基于注意力的模型之外，图神经网络、扩散模型和对比学习也被逐步引入 POI 序列建模。Li 等提出基于时空门控图网络的方法来强化复杂依赖建模 \cite{li2022atstggnn}；Qin 等通过扩散模型对用户空间偏好进行建模，为 POI 序列推荐提供了概率生成视角 \cite{qin2023diffpoi}；Qi 等则引入反事实序列合成与连续时间动态偏好建模，以缓解长序列场景下的数据稀疏与噪声问题 \cite{qi2024counterfactualpoi}。
实际上，无论采用何种序列建模框架，POI 表示本身仍是影响最终效果的重要基础。如果物品表示层难以体现地点间的语义与空间关联，再复杂的序列模型也难以充分释放其建模能力。

\subsection{生成式推荐与大语言模型}

近年来，大语言模型（Large Language Models, LLMs）和生成式检索（Generative Retrieval）的快速发展推动了推荐系统研究范式的重要转变。传统推荐系统通常遵循“召回—排序”的判别式框架，将推荐目标视为对候选集合中的项目进行打分与排序；而生成式推荐则试图将推荐问题重构为条件生成问题，使模型直接根据用户行为与上下文生成目标物品标识。Wang 等指出，生成式推荐代表了推荐系统从判别式建模向统一生成式建模演进的重要方向，其优势在于能够在统一 token 空间中联合处理用户历史、上下文信息与目标对象，并具备更强的开放性与表达能力 \cite{wang2023genrecsurvey}。Rajput 等提出 TIGER，将推荐任务转化为生成式检索问题，证明了通过结构化离散标识进行目标生成具有较强潜力 \cite{rajput2023tiger}。Li 等进一步对大语言模型驱动的生成式推荐进行了系统综述，指出 LLM 在处理复杂上下文、统一建模多源信息和提升可解释性方面展现出明显优势 \cite{li2023llmgenrecsurvey}。

围绕生成式推荐，研究者开始关注如何使语言模型更适配推荐任务本身。Hua 等讨论了推荐基础模型中的 item ID 索引问题，指出项目标识的设计直接影响模型的可学习性与泛化能力 \cite{hua2023index}；Ji 等提出 GenRec，尝试利用大语言模型完成更一般的生成式推荐任务 \cite{ji2024genrec}；Tan 等提出 IDGenRec，通过文本化 ID 学习强化大模型与推荐系统之间的对齐能力 \cite{tan2024idgenrec}；Wang 等进一步提出可学习的 item tokenization 方法，试图同时整合层次语义、协同信号和编码多样性，以获得更适合生成式推荐的项目表示 \cite{wang2024letter}，Li 等专门讨论了大语言模型在 Next POI 推荐中的应用前景，指出相较于传统纯数值建模方法，LLM 有望在更丰富的上下文理解和生成式预测方面发挥作用 \cite{li2024llm4poi}。这些研究共同说明，在生成式推荐框架下，模型性能不仅由用户序列建模能力决定，同时也与目标对象如何被编码为可生成的token密切相关。

% 在 POI 场景中，LLM 的引入尤其具有吸引力。一方面，用户历史访问序列、时间上下文和地点属性可以自然表示为结构化序列；另一方面，地点推荐往往需要综合考虑丰富的上下文约束，这与语言模型擅长处理复杂条件输入的特性相契合。Li 等专门讨论了大语言模型在 Next POI 推荐中的应用前景，指出相较于传统纯数值建模方法，LLM 有望在更丰富的上下文理解和生成式预测方面发挥作用 \cite{li2024llm4poi}。不过，这也带来一个新的问题：如果目标 POI 仍然使用无语义的随机 ID，那么即使语言模型具备强大能力，也难以从输入上下文中学习到稳定而自然的生成映射。由此，引出了 Semantic ID 与离散化物品表示研究的重要性。

\subsection{语义标识（Semantic ID）}

为了解决生成式推荐中随机原子 ID 难以学习、难以泛化的问题，研究者开始探索语义标识（Semantic ID）与更一般的离散化物品表示方法。其基本思想是先根据物品的连续表示学习结构化离散编码，使语义相似的对象在 token 空间中共享一定的层次结构，再将这些离散编码作为生成模型的目标输出。这样一来，模型不再需要直接记忆无意义的数字标签，而是能够在离散空间中利用对象间的潜在相似性。Rajput 等在 TIGER 中验证了残差量化式语义 ID 在生成式推荐中的有效性 \cite{rajput2023tiger}。随后，Hua 等对推荐基础模型中的 ID 索引策略进行了讨论，说明物品标识不仅是工程实现问题，更是影响生成模型学习效率的关键设计因素 \cite{hua2023index}。Wang 等提出可学习 item tokenization 方法，尝试在离散编码中融合层次语义、协同关系与码本多样性 \cite{wang2024letter}；Chen 等则进一步研究了通过自我改进机制优化 item tokenization 的可能性，反映出这一方向正在快速发展 \cite{chen2024selfimproveitemtoken}。

在面向推荐的大模型研究中，Semantic ID 的价值主要体现在两个方面。其一，它有助于缓解生成空间过大、目标难以学习的问题；其二，它能够在离散层面显式保留物品之间的相似性结构，从而增强模型的可泛化能力。围绕这一点，Hou 等研究了长语义 ID 的并行生成，说明随着目标对象复杂度增加，如何获得兼具表达性与可生成性的离散标识已成为关键问题 \cite{hou2025longsid}。对于 POI 场景，Wang 等提出 GNPR-SID，将 Semantic ID 引入 Next POI 推荐任务，首先利用 RQ-VAE 根据 POI 的属性与协同信息学习多级离散标识，再使用大语言模型根据历史访问 SID 序列预测下一兴趣点的 SID \cite{wang2025gnprsid}。这一工作表明，相比原始数字 ID，具有语义结构的离散表示更适合用于生成式 POI 推荐，也为后续研究提供了明确的技术路线。

% 然而，现有 Semantic ID 研究总体上仍更关注类别语义、文本信息和协同行为模式，而对 POI 任务中特有的空间结构信息考虑不足。对于一般物品推荐而言，这种以语义与协同为主的建模已能带来明显效果；但对于 POI 推荐而言，地点除了“是什么”之外，还天然具有“在哪里”的属性。如果 Semantic ID 的形成过程主要由类别、区域或协同访问信号驱动，那么最终学得的离散标识更可能反映语义聚类结果，而不一定能充分刻画真实地理邻近关系、区域差异以及用户活动半径约束。这也意味着，在 POI 场景下，Semantic ID 的设计仍有进一步引入地理信息的必要。

\subsection{POI 场景中的地理信息建模}

地理信息建模是 POI 推荐区别于一般推荐任务的关键所在。由于用户在现实世界中的移动行为受到活动半径、道路网络、区域功能分布和时间安排等多重因素制约，下一地点选择往往同时受到兴趣偏好与空间可达性的共同影响。因此，仅依赖类别语义或协同信号通常难以准确刻画用户真实决策过程。现有研究围绕地理因素的建模大致可以分为三类。第一类方法将经纬度、距离衰减或邻近关系作为附加数值特征直接引入模型；第二类方法在时空联合建模中，通过图结构、注意力机制或转移概率显式建模空间关系；第三类方法则进一步尝试将地理信息嵌入 POI 表示学习过程，使空间结构成为地点表征本身的一部分。

在具体研究中，Wang 等提出图增强时空网络，通过图嵌入方式显式刻画地理依赖，说明空间关系不应仅作为外部特征使用，而可以通过结构建模深度融入预测过程 \cite{wang2022graph}。Qin 等进一步对 POI 推荐中的地理效应进行解耦分析，指出地理偏置与真实兴趣偏好并不完全等价，若二者混合建模可能会影响推荐效果，因此有必要更细粒度地理解空间因素在推荐中的作用 \cite{qin2023disentanglegeo}。Pan 等提出 GeoCo，利用地理相关性增强 POI 表示学习，反映出学界已经开始关注“空间相关性本身是否能够作为 POI 表征的一部分”这一问题 \cite{pan2024geoco}。与此同时，Li 等提出面向区域感知的动态超图建模方法，通过双层交互同时建模点级和区域级关系，说明 Next POI 预测中的空间影响不仅体现在邻近 POI 之间，也体现在更高粒度的区域活动模式之中 \cite{li2025rehdm}。

从总体趋势看，现有工作已经充分证明地理信息在 POI 推荐中的重要性，但多数方法仍然是在预测阶段、排序阶段或图结构构建阶段使用空间信息，而较少真正让地理因素参与上游的离散语义表示学习过程。
% 换言之，现有研究更多回答了“如何让模型在预测时利用地理约束”，却较少回答“如何让地理信息直接影响 Semantic ID 的形成”。
对于本文所关注的生成式 Next POI 推荐任务，这恰恰是一个值得深入研究的切入点。

\section{本文研究内容与章节安排}
\label{sec:main_work}

基于上述研究背景与现状分析可以看出，Next POI 推荐任务已经从传统的时空建模逐步发展到结合大语言模型和生成式推荐的新阶段，但 POI 表示学习仍然是制约模型性能的关键因素之一。特别是在 GNPR-SID 这类基于 Semantic ID 的方法中，SID 的质量直接决定了后续生成模型的学习难度和预测效果。如果 SID 仅由类别、区域和协同信号学习得到，而缺乏对真实地理空间关系的充分建模，那么模型很难在输出阶段真正理解同类 POI 的空间差异和用户活动区域约束。

围绕这一问题，本文在 GNPR-SID 框架基础上开展研究，重点关注如何在 Semantic ID Construction 阶段引入地理信息增强表示，使地理信息能够在 POI 离散表示学习过程中发挥作用。本文首先对原始签到数据进行整理与清洗，构建用户访问序列和 POI 属性表；随后，围绕 POI 的类别名称和地理坐标构造地理信息增强表示，并利用 RQ-VAE 学习地理增强的 SID；在此基础上，进一步沿用 GNPR-SID 的生成式推荐框架，将学习得到的 SID 组织为结构化 token 序列，输入大语言模型进行监督微调，以实现 Next POI 推荐。

全文的组织结构如下。第二章介绍相关理论与技术基础，包括 POI 序列预测、残差量化式表示学习、大语言模型与参数高效微调，以及地理空间编码等与下文衔接的预备知识。第三章在 GNPR-SID 基础上阐述本文的改进动机、整体方案与实现要点。第四章给出实验设置、结果与分析。第五章总结全文并讨论后续工作方向。
