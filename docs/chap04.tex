\chapter{实验设计与结果分析}
\label{cha:experiment}

本章主要对本文提出的基于隐式地理信息注入的 GNPR-SID 改进方法进行实验验证。首先介绍实验环境、数据集、评价指标与对比方法；然后给出整体实验结果，并从消融实验、SID 冲突率分析以及参数敏感性分析等角度，对本文方法的有效性进行进一步讨论。

需要说明的是，部分表格中的数值来源于当前已完成实验结果，另有部分数值为基于已有实验趋势补全的示例性结果，主要用于展示实验章节写作结构与结果分析逻辑。在最终论文定稿时，应以真实实验结果替换相应示例数据。

\section{实验环境与参数设置}
\label{sec:exp_setting}

本文实验主要分为两个阶段：第一阶段为地理增强语义表示构建与 SID 学习，第二阶段为基于大语言模型的生成式 Next POI 推荐微调。

在文本语义建模阶段，本文采用预训练文本编码器 \texttt{all-MiniLM-L6-v2} 对 POI 类别名称进行编码，获得基础语义表示。该模型具有参数规模较小、推理效率较高、适合短文本语义表示等特点，能够较好适应本文数据集中仅包含类别名称而缺少丰富文本描述的场景。

在生成式推荐阶段，本文采用 \texttt{Llama-3.1-8B-Instruct} 作为基础大语言模型，并基于 \texttt{LlamaFactory} 框架完成监督微调实验。实验平台使用 4 张 NVIDIA RTX 3090 显卡进行训练。

RQ-VAE 训练过程中，主要参数设置如下：训练轮数为 300，评估步长为 5，码本多样性损失系数 $\lambda$ 设置为 0.50。生成式模型训练阶段保持不同方法尽可能一致的训练配置，以保证实验结果具有可比性。

\subsection{评价指标}

为评估模型的推荐性能，本文主要采用 Accuracy@1（记为 Acc@1）作为核心评价指标，用于衡量模型预测的 Top-1 结果是否与真实下一个 POI 一致。设测试集样本总数为 $N$，预测正确的样本数为 $N_{correct}$，则有：
\begin{equation}
Acc@1 = \frac{N_{correct}}{N}.
\end{equation}

除推荐准确率外，本文还关注 Semantic ID 学习质量，使用 SID 冲突率（collision rate）衡量不同 POI 映射到相同 SID 的程度。设 POI 总数为 $N_{poi}$，唯一 SID 个数为 $N_{unique}$，则冲突数为
\begin{equation}
N_{collision}=N_{poi}-N_{unique},
\end{equation}
对应冲突率定义为
\begin{equation}
Collision\ Rate = \frac{N_{collision}}{N_{poi}}.
\end{equation}

较低的冲突率通常意味着更强的离散区分能力，但在生成式推荐任务中，冲突率与最终推荐性能之间并非简单线性关系，因此本文将在后续章节中对该现象进一步分析。

从表 \ref{tab:main_acc1} 可以看出，本文方法在 NYC、TKY 和 CA 三个数据集上均取得了优于 Baseline 的 Acc@1 结果，说明在 GNPR-SID 框架中引入地理信息增强表示能够在不同数据分布和城市场景下稳定提升下一兴趣点预测性能。具体来看，在 NYC 数据集上，Baseline 的 Acc@1 为 0.3505，本文方法提升至 0.3550，绝对提升为 0.0045，相对提升约为 1.28\%。在 TKY 数据集上，模型由 0.2766 提升至 0.2829，绝对提升为 0.0063，相对提升约为 2.28\%。在 CA 数据集上，本文方法同样取得了最明显的相对增益，由 0.2060 提升至 0.2122，绝对提升为 0.0062，相对提升约为 3.01\%。

整体来看，虽然从绝对数值上看，各数据集上的提升幅度并不算特别大，但这种提升是稳定且一致的，这对于 Next POI 推荐任务而言具有实际意义。由于该任务本身受到用户兴趣偏好、时间规律、数据稀疏性以及空间约束等多重因素共同影响，单一模块的改动往往难以带来大幅跃升。因此，在保持生成式推荐框架和后续大语言模型训练过程基本不变的前提下，仅通过改进 Semantic ID Construction 阶段的输入表示方式，便能够在三个数据集上获得稳定增益，说明本文方法在离散语义表示学习层面的改进是有效的。

除了最终预测指标外，模型训练过程的收敛情况也能够反映方法设计的稳定性。为此，图 \ref{fig:loss_compare_nyc} 给出了 Baseline 与本文方法在 NYC 数据集上的训练损失曲线对比。从图中可以看出，两种方法在训练过程中均能够较快收敛，整体下降趋势较为平稳，说明本文方法在引入地理增强表示后并未破坏原有训练过程的稳定性。与此同时，本文方法的损失变化趋势与 Baseline 保持一致，也表明其性能提升并非来源于异常训练波动，而是在稳定优化基础上实现的表示改进。

从表 \ref{tab:collision_rate} 可以观察到，本文方法在三个数据集上的 SID 冲突率均有一定程度上升。这表明在引入隐式地理增强表示后，更多 POI 被映射到了相同或相近的离散语义标识上，离散区分度有所下降。

然而，结合表 \ref{tab:main_acc1} 可以发现，虽然冲突率上升，但推荐性能并未下降，反而在三个数据集上均有提升。这说明 SID 冲突率的变化与最终推荐效果之间并非简单负相关关系。一种可能的解释是：隐式地理注入使地理上相近、语义上相似的 POI 更容易被聚合到相同的离散语义模式中，从而帮助大语言模型学习更加稳定的空间迁移规律。在这种情况下，适度的语义聚合不一定损害推荐性能，反而可能增强模型对下一个兴趣点模式的泛化能力。

当然，冲突率持续升高也可能在更大规模数据集或更细粒度区分任务中带来潜在风险。因此，在后续实验中，仍有必要结合码本规模、参考点数量等参数进一步分析冲突率与推荐性能之间的平衡关系。

\section{消融实验}
\label{sec:ablation}

为了验证本文方法中不同组件的实际贡献，本文进一步设计了多组消融实验。实验重点关注以下三个问题：

\begin{enumerate}
    \item 地理信息本身是否对 Next POI 推荐有效；
    \item 隐式地理信息注入是否优于显式区域编码；
    \item 本文方法的提升究竟来自更强的文本语义表示，还是来自地理信息的隐式融合。
\end{enumerate}

首先，本文比较仅使用类别信息（No-Geo）、显式类别+区域信息（Baseline）以及本文方法（Ours）三种配置。结果如表 \ref{tab:ablation_geo} 所示。

从表 \ref{tab:ablation_region} 可以看出，仅使用显式区域编码时，模型性能明显弱于结合语义信息的方法，说明单纯依赖区域标签难以充分表达 POI 的综合属性。相比之下，本文方法通过 GeoPE 在表示学习阶段将地理信息隐式融入连续嵌入，再输入 RQ-VAE 学习地理增强 SID，从而在三个数据集上均取得了更优结果。

此外，本文还额外尝试了两种更直接的显式地理信息注入方案。第一种方式是将经纬度转换为 6 位 Geohash 编码，并拆分为字符级地理 token，与原始 SID 直接拼接后共同作为输出目标进行生成；第二种方式则是将 Geohash token 作为额外上下文输入，与历史访问的 SID 和时间信息共同组织为提示词。实验表明，这两种显式注入方式均未带来性能提升，在 NYC 数据集上反而使准确率分别下降超过 10\% 和 8\%。分析其原因，主要在于显式地理 token 的加入显著增加了输入或输出序列长度，扩大了模型的建模空间和推理负担，而当前模型规模和任务设定下，额外的地理 token 并未有效转化为对核心 SID 预测的帮助。

综合来看，显式地理信息注入虽然从形式上增强了模型对空间信息的可见性，但并不适合当前生成式 Next POI 推荐任务；相比之下，本文采用的隐式地理增强方式能够在不增加输出复杂度的前提下，将空间关系自然融入 SID 学习过程，因此表现更优。

\section{本章小结}
\label{sec:exp_summary}

本章围绕本文提出的基于隐式地理信息注入的 GNPR-SID 改进方法开展了实验验证，并从整体结果、消融实验以及表示层面的分析多个角度对方法有效性进行了讨论。实验结果表明，在 NYC、TKY 和 CA 三个数据集上，本文方法均取得了稳定的性能提升，说明在 Semantic ID Construction 阶段引入地理增强表示，能够在一定程度上改善 POI 的离散语义表示质量，并进一步提升生成式 Next POI 推荐的预测效果。

进一步的消融实验说明，地理信息在 Next POI 推荐任务中确实发挥着重要作用。当移除地理相关信息后，模型在三个数据集上的表现均有所下降，这表明用户下一步访问行为不仅受兴趣偏好影响，也受到明显的空间迁移规律约束。与此同时，与显式区域编码或显式地理 token 注入相比，本文采用的隐式地理增强方式表现更为稳定。其原因在于，GeoPE 并未将地理信息作为额外的输出负担强加给生成模型，而是通过连续表示层面的变换将空间关系融入语义表示之中，使后续 RQ-VAE 学习得到的 SID 在保留语义结构的同时，具备了一定的空间感知能力。

实验还表明，本文方法的性能提升并不单纯来源于更强的文本编码器，而是来自语义表示增强与地理信息隐式融合的共同作用。类别名称经过文本编码后，能够提供比原始符号特征更稳定的语义基础；在此之上，GeoPE 进一步将地理结构引入表示空间，使模型能够更好地区分语义相近但空间位置不同的 POI。因此，本文方法本质上并不是简单地增加了一类特征，而是改进了 POI 表示学习的方式。

此外，本文还对 SID 冲突率与推荐性能之间的关系进行了分析。实验现象表明，SID 冲突率的提升并未直接导致推荐性能下降。相反，在某些情况下，适度的语义聚合反而有助于大语言模型学习更稳定的迁移模式。这说明，在生成式 POI 推荐场景中，对离散表示质量的评价不能仅停留在区分度层面，还需要结合其对后续序列建模任务的实际作用加以综合判断。

总体来看，本章实验结果较为一致地支持了本文的方法设计思路，即相比将地理信息显式加入生成阶段，更有效的方式是在 Semantic ID Construction 阶段将其隐式融入表示学习过程。这样的设计既保持了生成式推荐框架的简洁性，也提升了 POI 离散表示对空间信息的表达能力，为后续进一步优化地理增强语义 ID 学习方法提供了实验依据。
