\section{实验环境与参数设置}
\label{sec:exp_setting}

本文实验主要分为两个阶段：第一阶段为地理增强语义表示构建与 SID 学习，第二阶段为基于大语言模型的生成式 Next POI 推荐微调。

在文本语义建模阶段，本文采用预训练文本编码器 \texttt{all-MiniLM-L6-v2} 对 POI 类别名称进行编码，获得基础语义表示。该模型具有参数规模较小、推理效率较高、适合短文本语义表示等特点，能够较好适应本文数据集中仅包含类别名称而缺少丰富文本描述的场景。

在生成式推荐阶段，本文采用 \texttt{Llama-3.1-8B-Instruct} 作为基础大语言模型，并基于 \texttt{LlamaFactory} 框架完成监督微调实验。实验平台使用 4 张 NVIDIA RTX 3090 显卡进行训练。

RQ-VAE 训练过程中，主要参数设置如下：训练轮数为 300，评估步长为 5，码本多样性损失系数 $\lambda$ 设置为 0.50。生成式模型训练阶段保持不同方法尽可能一致的训练配置，以保证实验结果具有可比性。

\subsection{评价指标}

为评估模型的推荐性能，本文主要采用 Accuracy@1（记为 Acc@1）作为核心评价指标，用于衡量模型预测的 Top-1 结果是否与真实下一个 POI 一致。设测试集样本总数为 $N$，预测正确的样本数为 $N_{correct}$，则有：
\begin{equation}
Acc@1 = \frac{N_{correct}}{N}.
\end{equation}

除推荐准确率外，本文还关注 Semantic ID 学习质量，使用 SID 冲突率（collision rate）衡量不同 POI 映射到相同 SID 的程度。设 POI 总数为 $N_{poi}$，唯一 SID 个数为 $N_{unique}$，则冲突数为
\begin{equation}
N_{collision}=N_{poi}-N_{unique},
\end{equation}
对应冲突率定义为
\begin{equation}
Collision\ Rate = \frac{N_{collision}}{N_{poi}}.
\end{equation}

较低的冲突率通常意味着更强的离散区分能力，但在生成式推荐任务中，冲突率与最终推荐性能之间并非简单线性关系，因此本文将在后续章节中对该现象进一步分析。

整体来看，虽然从绝对数值上看，各数据集上的提升幅度并不算特别大，但这种提升是稳定且一致的，这对于 Next POI 推荐任务而言具有实际意义。由于该任务本身受到用户兴趣偏好、时间规律、数据稀疏性以及空间约束等多重因素共同影响，单一模块的改动往往难以带来大幅跃升。因此，在保持生成式推荐框架和后续大语言模型训练过程基本不变的前提下，仅通过改进 Semantic ID Construction 阶段的输入表示方式，便能够在三个数据集上获得稳定增益，说明本文方法在离散语义表示学习层面的改进是有效的。

从表 \ref{tab:collision_rate} 可以观察到，本文方法在三个数据集上的 SID 冲突率均有一定程度上升。这表明在引入隐式地理增强表示后，更多 POI 被映射到了相同或相近的离散语义标识上，离散区分度有所下降。

然而，结合表 \ref{tab:main_acc1} 可以发现，虽然冲突率上升，但推荐性能并未下降，反而在三个数据集上均有提升。这说明 SID 冲突率的变化与最终推荐效果之间并非简单负相关关系。一种可能的解释是：隐式地理注入使地理上相近、语义上相似的 POI 更容易被聚合到相同的离散语义模式中，从而帮助大语言模型学习更加稳定的空间迁移规律。在这种情况下，适度的语义聚合不一定损害推荐性能，反而可能增强模型对下一个兴趣点模式的泛化能力。

当然，冲突率持续升高也可能在更大规模数据集或更细粒度区分任务中带来潜在风险。因此，在后续实验中，仍有必要结合码本规模、参考点数量等参数进一步分析冲突率与推荐性能之间的平衡关系。

\section{消融实验}
\label{sec:ablation}

为了验证本文方法中不同组件的实际贡献，本文进一步设计了多组消融实验。实验重点关注以下三个问题：

\begin{enumerate}
    \item 地理信息本身是否对 Next POI 推荐有效；
    \item 隐式地理信息注入是否优于显式区域编码；
    \item 本文方法的提升究竟来自更强的文本语义表示，还是来自地理信息的隐式融合。
\end{enumerate}

\section{实验结果讨论}
\label{sec:discussion}

综合整体实验与消融实验结果，可以得到以下几点结论：

\begin{enumerate}
    \item \textbf{地理信息对 Next POI 推荐具有显著作用。} 当移除地理信息后，模型性能在三个数据集上均有所下降，说明用户的空间迁移规律是决定下一兴趣点的重要因素。
    
    \item \textbf{隐式地理信息注入优于显式区域编码。} 相比使用 Geohash 等显式区域标签，本文提出的 GeoPE 方法通过旋转位置编码将地理关系隐式融入语义表示空间，从而获得更优的离散语义 ID 与更高的推荐准确率。
    
    \item \textbf{本文方法的提升来源于“更强语义表示 + 空间隐式融合”的共同作用。} 文本编码器本身能够提升类别语义表示质量，而 GeoPE 进一步增强了模型对空间结构的感知能力，两者结合后取得最佳效果。
    
    \item \textbf{SID 冲突率提升并未导致推荐性能下降。} 相反，适度的语义聚合可能帮助大语言模型学习更稳定的迁移模式。这表明在生成式 POI 推荐任务中，应综合考察离散表示区分度与序列建模效果，而不能仅以冲突率高低评价 SID 质量。
\end{enumerate}

\section{本章小结}
\label{sec:exp_summary}

本章对本文提出的基于隐式地理信息注入的 GNPR-SID 改进方法进行了实验验证。实验结果表明，本文方法在 NYC、TKY 和 CA 三个数据集上均取得了一定程度的性能提升，验证了隐式地理增强表示在生成式 Next POI 推荐任务中的有效性。进一步的消融实验表明，地理信息本身对推荐任务具有重要作用，而相比显式区域编码，GeoPE 能够以更加自然的方式将空间关系融入语义表示空间，从而生成更具判别性和建模能力的语义 ID。

同时，本文还分析了 SID 冲突率变化与推荐性能之间的关系，说明在生成式推荐场景下，适度的语义聚合可能有助于提升模型对空间迁移模式的学习能力。总体而言，实验结果支持了本文提出的方法设计动机，并为后续进一步优化 Semantic ID Construction 模块提供了实验依据。
