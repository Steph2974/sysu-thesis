\chapter{基于隐式地理信息注入的 POI 语义表示构建方法}
\label{cha:method}

本章围绕本文提出的基于隐式地理信息注入的 POI 语义表示构建方法展开介绍。本文的研究工作以 GNPR-SID 框架为基础，面向 Next POI Recommendation 任务，对其 Semantic ID Construction 模块进行改进。原始 GNPR-SID 方法通过构建具有语义结构的离散标识，并结合大语言模型完成下一兴趣点生成式预测，在生成式推荐任务中展现出较好的性能。然而，原方法对地理信息的建模仍然较为粗粒度，尚未充分刻画 POI 之间真实的空间关系。

针对上述问题，本文从地理信息注入方式出发，对原始方法进行分析与改进。本文首先从建模复杂度与训练目标结构等角度，对显式地理信息注入与隐式地理信息注入两类方案进行了对比讨论。初步分析表明，将地理信息直接并入大语言模型输入或输出可能显著增加序列长度与生成难度，且未必与核心 SID 预测形成稳定协同；该类判断仍需结合具体数据与任务由实验加以检验（见第 \ref{cha:experiment} 章）。在此基础上，本文选择在 Semantic ID Construction 阶段引入隐式地理信息注入机制，利用文本编码器与 GeoPE（Geographical Positional Encoding）构造融合语义与空间信息的连续 POI 表示，再结合 RQ-VAE 学习地理增强的语义 ID，从而提升 POI 表示的空间感知能力，并为后续生成式推荐提供更高质量的离散表示。

\section{问题分析与方法设计动机}
\label{sec:problem_analysis}

POI 推荐任务与一般序列推荐任务的重要区别在于，POI 本身具有显著的空间属性。用户下一步访问的兴趣点不仅受到历史兴趣偏好的影响，还与当前位置、空间可达性、活动半径以及城市功能区域分布密切相关。因此，如果离散语义标识仅由类别、时间和协同信息等特征学习得到，而没有充分建模真实地理空间关系，那么最终得到的 SID 对 POI 的表达能力仍然是不完整的。

在 GNPR-SID 原始框架中，空间信息主要通过区域级地理编码以特征拼接的方式加入连续表示。这种做法虽然能够提供一定的区域先验，但仍存在以下两个问题。

第一，区域编码本质上更接近符号化标签，只能反映粗粒度的空间位置，难以刻画 POI 之间更加细粒度的地理差异。例如，同属某一区域内的两个 POI 可能在真实距离、方向关系和功能邻近性上存在明显差别，但这些差别在单一的区域编码中难以体现。

第二，地理信息与语义信息之间主要表现为简单拼接关系，而不是在表示学习过程中形成统一的潜在空间结构。换言之，原始方法虽然“加入了地理信息”，但这种加入方式并未真正改变 POI 语义表示的内部几何结构，因此空间信息对最终 SID 学习结果的影响仍然有限。

基于上述分析，本文的核心思路是：不再将地理信息视为一个附加特征，而是将其视为一种能够作用于语义表示的变换机制，使空间关系在表示学习阶段直接影响 POI 的连续表示构造。围绕这一目标，本文先后探索了显式地理信息注入与隐式地理信息注入两类方案，并最终选择在 Semantic ID Construction 阶段引入隐式空间增强表示。

\section{显式地理信息注入方案及其局限性}
\label{sec:explicit_geo_attempt}

在方法探索初期，本文首先尝试通过显式方式将地理信息注入 Generative POI Recommendation 模块。该方案的基本出发点在于：既然 GNPR-SID 最终通过大语言模型生成下一兴趣点的 SID，那么可以进一步将地理位置编码组织为显式 token，并直接注入到历史访问序列提示词中，甚至将其设计为输出目标的一部分，从而使模型在生成下一 POI 时能够显式感知空间信息。

具体而言，本文考虑采用 Geohash 等离散地理编码方式，将经纬度映射为统一的地理 token 序列，并与原始 SID 共同构成输入或输出表示。在这种设计下，历史访问序列中的每个 POI 不再仅由其语义 SID 表示，而是扩展为“语义 token + 地理 token”的联合表示。相应地，模型输出也可设计为目标 POI 的地理 token 与 SID token 的联合结果。

从直觉上看，该方案能够使地理信息在 token 层面被大语言模型直接“看见”，似乎有助于模型学习 POI 之间的区域相似性与空间迁移模式。然而，从生成式建模机制出发，该方案至少面临以下几方面风险或代价。
首先，显式地理 token 的引入会直接增加输入与输出序列长度。对于自回归生成模型而言，序列长度的增加意味着训练成本和推理成本同步上升，尤其是在历史访问序列本身已经较长的情况下，这种额外开销更加明显。
其次，当地理 token 与 SID token 并列出现在输出端时，模型事实上需要同时完成地理预测与语义预测两个子任务，导致训练目标被分散。当地理 token 与核心 SID 判别之间难以形成稳定、直接的协同关系时，这种联合建模存在削弱模型对核心推荐目标关注的风险。
最后，显式地理信息虽然提高了空间因素在模型输入输出中的可见性，但从机制上看，其能否转化为对 SID 预测的有效帮助并不显然：输出空间扩大与辅助目标增加，均可能提高优化难度。上述判断主要基于对任务结构与自回归生成特性的定性分析；显式方案在本文设定下的实际效果及与隐式方案的对比，见第 \ref{cha:experiment} 章的实验。

综合上述分析，显式地理信息注入在形式上更为直接，但可能显著增加生成式建模复杂度，且其相对收益需由实验验证。因此，本文将后续重点放在表示学习阶段的隐式地理信息注入，并在第 \ref{cha:experiment} 章通过实验进一步讨论显式 token 方案的表现。

\section{基于 GeoPE 的隐式地理增强表示构建}
\label{sec:implicit_geope}

与显式注入方案不同，隐式地理信息注入更关注在连续表示层面建模空间关系，并通过连续表示的改变影响最终 SID 的学习结果。本文借鉴基于旋转位置编码的空间建模思想，在 POI 表示学习阶段引入 GeoPE（Geographical Positional Encoding）机制，使地理信息能够以结构化方式嵌入潜在表示空间，从而得到同时包含语义信息与空间信息的连续 POI 表示。

% 从 Semantic ID Construction 的目标来看，本文期望最终学习得到的 SID 至少具备以下三方面性质。第一，\textbf{语义表达性}，即离散标识应尽可能保留 POI 的类别语义信息，使同类或功能相近的 POI 在表示空间中具有可辨识的相似性。第二，\textbf{空间敏感性}，即 SID 不仅反映 POI 的语义属性，还能够区分其在真实地理空间中的相对位置差异，使地理邻近性与空间结构能够通过离散标识间接体现出来。第三，\textbf{层次可区分性}，即语义相近或空间关系相近的 POI 应倾向于共享更长的 SID 前缀，而差异较大的 POI 则在更高层级上被区分开来。围绕这三个目标，本文先构造语义--空间联合连续表示，再通过 RQ-VAE 将其映射到具有层次结构的离散语义 ID 空间。

\subsection{基础语义表示构建}

设某一 POI 的文本属性记为 $A_p$。在本文所使用的数据集中，POI 缺少丰富的自然语言描述信息，能够直接利用的文本属性主要为类别名称。因此，本文首先采用轻量级文本编码器对类别属性进行编码，得到基础 POI 语义表示：
\begin{equation}
x_p = f_{\mathrm{text}}(A_p),
\end{equation}
其中，$x_p \in \mathbb{R}^{D}$ 表示 POI 的基础语义嵌入，$f_{\mathrm{text}}(\cdot)$ 表示文本编码器。本文具体采用预训练模型 \texttt{all-MiniLM-L6-v2} 作为文本编码器，其输出维度为 $D=384$。之所以选择该模型，一方面是因为类别名称文本较短，使用轻量模型即可获得较稳定的语义表示；另一方面，较小的编码器规模也有助于控制整体训练成本。

\subsection{参考点集合构建}

为了建模空间依赖关系，本文不直接对经纬度数值进行拼接，而是首先基于所有 POI 的空间分布构造若干参考点。具体地，设所有 POI 的经纬度坐标集合为 $\mathcal{L}=\{l_p\}_{p=1}^{N}$，其中 $l_p=(lat_p, lon_p)$。本文采用 K-Means 聚类方法对所有 POI 的坐标进行聚类，得到 $\Omega$ 个参考点：
\begin{equation}
R = \{R_1, R_2, \dots, R_{\Omega}\}.
\end{equation}
其中，每个参考点 $R_{\omega}$ 可视为一个空间密度中心，用于刻画城市空间中的典型区域结构。通过引入多个参考点，本文可以从相对位置而非绝对坐标的角度对 POI 的空间关系进行建模。

\subsection{方位角计算}

对于任意 POI $p$ 及参考点 $R_{\omega}$，本文计算两者之间的相对方位角 $\theta_{p,\omega}$，用于表示 POI 相对于该参考点的方向关系。记参考点坐标为 $R_{\omega}=(lat_{\omega}, lon_{\omega})$，则可通过如下形式计算对应的方位角：
\begin{equation}
\theta_{p,\omega} = \mathrm{atan2}(lat_p-lat_{\omega},\, lon_p-lon_{\omega}).
\end{equation}

对于每一个 POI，关于全部 $\Omega$ 个参考点可以得到一组方位角
\begin{equation}
\Theta_p = \{\theta_{p,1}, \theta_{p,2}, \dots, \theta_{p,\Omega}\}.
\end{equation}
该角度集合从几何上刻画了 POI 相对于多个空间锚点的方向关系，因而可以视为对其真实位置的一种隐式表示。与直接使用绝对经纬度不同，这种相对位置建模方式更强调 POI 在整体空间结构中的关系信息。

需要指出的是，单个参考点下的方位角主要表达的是相对方位（“朝向”），对欧氏距离并非直接可逆：若两个 POI 相对某一参考点近似共线同向但远近不同，仅凭该角度难以区分。为此，本文采用多参考点联合编码：不同锚点下的 $\theta_{p,\omega}$ 共同作用于嵌入的不同子块，使角度向量 $\Theta_p$ 对平面上的相对位置形成更强的约束，从而在一定程度上缓解单一角度表示的歧义；更细粒度的距离差异则仍部分依赖后续 RQ-VAE 的连续表示学习与量化过程加以吸收。若参考点数量偏少或其空间分布与数据分布不匹配，角度编码仍可能出现近似冲突，这是当前 GeoPE 设计的一个局限。

\subsection{基于旋转编码的空间信息注入}

在获得基础语义表示 $x_p$ 与对应的方位角集合 $\Theta_p$ 后，本文采用旋转变换的方式将地理信息注入语义向量。具体而言，首先将基础嵌入 $x_p$ 划分为 $\Omega$ 个子向量块：
\begin{equation}
x_p = [x_p^{(1)}; x_p^{(2)}; \dots; x_p^{(\Omega)}],
\end{equation}
其中，$x_p^{(\omega)}$ 表示第 $\omega$ 个子向量块，对应参考点 $R_{\omega}$。为便于进行二维旋转操作，要求每个子块的维度为偶数；当 $D$ 不能被 $\Omega$ 整除时，可通过线性投影或零填充方式将其映射到合适维度。

随后，对于第 $\omega$ 个子向量块中的相邻维度对，施加由方位角 $\theta_{p,\omega}$ 控制的二维旋转。设该子块中的某一对相邻维度为 $(x_{2i}^{(\omega)}, x_{2i+1}^{(\omega)})$，则其旋转结果表示为
\begin{equation}
\begin{bmatrix}
\hat{x}_{2i}^{(\omega)} \\
\hat{x}_{2i+1}^{(\omega)}
\end{bmatrix}
=
\begin{bmatrix}
\cos \theta_{p,\omega} & -\sin \theta_{p,\omega} \\
\sin \theta_{p,\omega} & \cos \theta_{p,\omega}
\end{bmatrix}
\begin{bmatrix}
x_{2i}^{(\omega)} \\
x_{2i+1}^{(\omega)}
\end{bmatrix}.
\end{equation}

对全部子向量块完成旋转操作后，将其重新拼接，得到最终的地理增强连续表示：
\begin{equation}
x_p' = \mathrm{GeoPE}(x_p, l_p).
\end{equation}

通过上述构造，原本仅包含类别语义信息的基础表示 $x_p$ 被映射为同时融合语义信息与空间信息的增强表示 $x_p'$。与直接拼接经纬度或区域编码的方式相比，GeoPE 并不将地理信息作为外部附加特征，而是通过对语义向量内部结构施加几何变换，使空间差异成为潜在表示的一部分。在理想情况下，若多参考点下的角度组合能够区分不同位置，则即使两个 POI 属于相同类别，其增强表示仍可能因 $\Theta_p$ 的差异而区分开来，从而提升表示的空间判别能力。

\section{基于地理增强表示的语义 ID 学习}
\label{sec:geo_sid_learning}

在获得地理增强表示 $x_p'$ 后，本文将其作为 RQ-VAE 的输入，用于学习具备空间感知能力的离散语义标识。与原始 GNPR-SID 采用多源显式特征拼接表示
\begin{equation}
p_e = \mathrm{concat}(c, r, t, c_u)
\end{equation}
不同，本文更加关注 POI 静态属性中的语义--空间联合表达，因此采用由文本编码器与 GeoPE 构造得到的连续表示 $x_p'$ 作为量化输入。这样做的目的是使后续离散化过程能够直接作用于已经融合地理信息的潜在表示，从而使学习得到的 SID 不仅能够表达语义相似性，也能够反映空间差异。

设编码器对地理增强表示的输出为
\begin{equation}
z_p = \mathrm{Enc}(x_p'),
\end{equation}
其中，$\mathrm{Enc}(\cdot)$ 表示编码器，它将输入的地理增强表示映射到更适于量化的潜在空间。为了将连续潜在向量离散化，本文在 RQ-VAE 中构造包含 $L$ 层量化码本的层次化离散空间。记第 $l$ 层码本为
\begin{equation}
\mathcal{B}^{(l)}=\{e_1^{(l)},e_2^{(l)},\dots,e_K^{(l)}\},
\end{equation}
其中 $e_k^{(l)} \in \mathbb{R}^{d}$ 表示第 $l$ 层码本中的第 $k$ 个 codeword，$K$ 表示每层码本的大小。RQ-VAE 采用逐层残差量化的方式逼近潜在表示。首先令初始残差为
\begin{equation}
r_p^{(0)} = z_p,
\end{equation}
随后在第 $l$ 层码本中选择与当前残差最接近的 codeword：
\begin{equation}
q_p^{(l)}=\arg\min_{e \in \mathcal{B}^{(l)}} \| r_p^{(l-1)}-e \|_2^2,
\end{equation}
并更新该层后的残差为
\begin{equation}
r_p^{(l)} = r_p^{(l-1)} - q_p^{(l)}.
\end{equation}

通过上述递归量化过程，模型能够以由粗到细的方式逐步逼近原始潜在表示：前层码本主要捕获较为粗粒度的全局语义与空间结构，后层码本则进一步补充细粒度差异。因此，若两个 POI 在语义属性和空间位置上较为接近，则其在前几层量化过程中更可能选择相同或相近的 codeword，从而共享更长的 SID 前缀，这也是 Semantic ID 具有层次相似性的重要来源。

在完成 $L$ 层量化后，可将所有层中选中的 codeword 向量求和，得到量化后的潜在表示：
\begin{equation}
\hat{z}_p = \sum_{l=1}^{L} q_p^{(l)}.
\end{equation}
随后，解码器根据量化表示重建输入向量：
\begin{equation}
\hat{x}_p' = \mathrm{Dec}(\hat{z}_p),
\end{equation}
其中 $\mathrm{Dec}(\cdot)$ 表示解码器。重建过程的作用在于约束离散化结果尽可能保留原始地理增强表示中的主要语义和空间信息，从而避免量化过程造成过多信息损失。若量化层数为 $L$，则每个 POI 最终对应一个由多级索引组成的离散表示：
\begin{equation}
SID(p) = [s_1, s_2, \dots, s_L].
\end{equation}
其中 $s_l=\mathrm{index}(q_p^{(l)})$ 表示第 $l$ 层所选 codeword 在对应码本中的索引。可见，最终的 SID 本质上是潜在表示在多层离散码本中的路径编码，而不是一个随机分配的整数编号。

由于输入表示 $x_p'$ 中已经隐式融入了地理信息，因此此处学习得到的 SID 不再是仅由语义属性主导的离散标识，而是一种兼具语义感知与空间感知能力的离散语义 ID。该设计使地理信息在不增加输出复杂度的前提下，直接参与了离散标识的学习过程。

在训练过程中，本文仍采用重建损失、量化损失与多样性损失共同优化 RQ-VAE，其总损失函数可表示为
\begin{equation}
\mathcal{L}_{total} = \mathcal{L}_{recon} + \mu \mathcal{L}_{quant} + \lambda \mathcal{L}_{div}.
\end{equation}
其中，$\mathcal{L}_{recon}$ 用于保证量化后的表示仍能够保留原始输入中的主要信息；$\mathcal{L}_{quant}$ 用于稳定编码器输出与量化码本之间的映射关系；$\mathcal{L}_{div}$ 用于鼓励码本被更加均衡地使用，防止不同样本过度聚集于少量 codeword 上。

需要说明的是，本文方法并不要求模型显式生成地理标识，而是让地理信息通过连续表示构造和量化学习过程隐式体现在 SID 中。该设计的关键优势在于，一方面保持了生成目标形式的统一性，避免了显式地理 token 带来的输出空间膨胀；另一方面又能够使离散语义标识真实受益于地理增强表示，从而提升后续生成式推荐的表示基础。

\section{地理增强 SID 在生成式 POI 推荐中的应用}
\label{sec:geo_sid_application}

在完成地理增强 SID 的学习后，本文沿用 GNPR-SID 的生成式推荐框架，将每个 POI 对应的 SID 组织为结构化 token 序列，并用于构造用户历史访问序列的输入提示。设 POI 到语义 ID 的映射函数记为
\begin{equation}
\mathcal{F}: p \mapsto SID(p),
\end{equation}
则用户历史访问序列中的每个 POI 均可被替换为其对应的 SID token 序列。设某一 POI 的 SID 表示为
\begin{equation}
SID(p) = [s_1, s_2, \dots, s_L],
\end{equation}
则其对应的 token 表示可记为
\begin{equation}
T(p)=\langle a_{s_1}\rangle \langle b_{s_2}\rangle \cdots \langle l_{s_L}\rangle.
\end{equation}

对于用户 $u$ 的历史访问序列
\begin{equation}
H_u=\{(p_1,t_1),(p_2,t_2),\dots,(p_n,t_n)\},
\end{equation}
可进一步映射为语义 ID 序列
\begin{equation}
S_u=\{(SID(p_1),t_1),(SID(p_2),t_2),\dots,(SID(p_n),t_n)\}.
\end{equation}

在此基础上，本文将其组织为如下形式的输入提示：
\begin{quote}
\small
\texttt{user\_<uid> visited: <SID> at [time], ..., visited <SID> at [time]. When [time] user\_<uid> is likely to visit:}
\end{quote}

与原始 GNPR-SID 的区别在于，此处的 \texttt{<SID>} 不再仅来自类别、区域与协同特征构建得到的离散表示，而是来源于地理增强表示 $x_p'$ 经 RQ-VAE 学习得到的地理增强 SID。也就是说，尽管生成式模型在输出端仍然只预测统一形式的 SID 序列，但其预测目标本身已经隐含了更多的地理空间信息。
对于大语言模型而言，输入端的 token 已从粗粒度语义，转变为语义和空间联合的形式，因此模型学习到的下一 POI 预测规则也会相应包含更强的空间判别能力。

在训练阶段，本文采用监督微调（Supervised Fine-Tuning, SFT）的方式对大语言模型进行训练，使其能够根据用户历史访问行为与时间上下文生成下一兴趣点的 SID。具体而言，以用户过去的 SID 访问序列及其时间信息作为输入，以真实下一次访问 POI 对应的 SID 作为监督目标，从而构建生成式训练样本。由于训练目标仍保持为统一形式的 SID 序列，本文方法无需额外扩展生成端词表结构，也不需要让模型同时预测额外的地理 token，因此能够在保持原有生成框架稳定性的同时，将地理信息的收益前移到离散表示学习阶段。

在推理阶段，模型输出的 SID 再通过查表映射回具体的 POI，从而完成 Next POI Recommendation 任务。若多个候选 POI 在高层 SID 前缀上相同，则模型实际上会优先依据更细粒度的后续 token 进行区分；因此，地理增强 SID 的层次化结构不仅有助于表示语义相近性，也有助于生成式模型逐步缩小候选空间，提高预测的可解释性。

由此可见，本文提出的方法并未改变 GNPR-SID 生成式推荐模块的整体建模方式，而是通过优化 Semantic ID Construction 阶段的输入表示，使后续生成式模型能够在不增加额外输出负担的前提下，利用更具空间判别能力的离散标识完成推荐任务。这也体现了本文方法的核心特征，即在保持原有生成框架稳定性的基础上，通过前端表示学习改进提升整体推荐性能。

\section{本章小结}
\label{sec:method_summary}

本章围绕本文提出的基于隐式地理信息注入的 POI 语义表示构建方法进行了系统介绍。首先，结合问题分析说明了 GNPR-SID 的基本思路（Semantic ID Construction 与 Generative POI Recommendation 两阶段）及其在地理信息建模方面存在的不足。随后，本文从机制层面对显式地理信息注入进行了讨论，说明将地理信息作为额外 token 注入生成模型虽然形式直观，但可能显著增加序列长度与输出复杂度，其相对优劣需结合实验进一步判断（见第 \ref{cha:experiment} 章）。

在此基础上，本文提出了一种基于 GeoPE 的隐式地理信息注入方法。该方法首先利用轻量文本编码器对 POI 类别属性进行编码，得到基础语义表示；随后，基于多个参考点与 POI 之间的相对方位角，对语义向量进行分块旋转，从而构造同时融合语义信息与空间信息的连续表示。在此基础上，本文进一步借助 RQ-VAE 对地理增强表示进行多层残差量化，使 Semantic ID 的学习过程能够直接作用于语义--空间联合潜在空间，进而得到兼具语义表达能力、空间敏感能力与层次结构特征的地理增强 SID。

最后，本文说明了如何将地理增强 SID 无缝接入 GNPR-SID 的生成式推荐框架。与显式地理 token 注入方案不同，本文并不改变大语言模型的输出形式，而是通过优化前端离散表示学习过程，使生成模型在维持原有建模范式与输出复杂度不变的条件下，获得更具空间判别能力的目标表示，为后续实验中的性能提升奠定了方法基础。
