\chapter{基于隐式地理信息注入的 POI 语义表示构建方法}
\label{cha:method}

本章围绕本文提出的基于隐式地理信息注入的 POI 语义表示构建方法展开介绍。本文的研究工作以 GNPR-SID 框架为基础，面向 Next POI Recommendation 任务，对其 Semantic ID Construction 模块进行改进。原始 GNPR-SID 方法通过构建具有语义结构的离散标识，并结合大语言模型完成下一兴趣点生成式预测，在生成式推荐任务中展现出较好的性能。然而，原方法对地理信息的建模仍然较为粗粒度，尚未充分刻画 POI 之间真实的空间关系。

针对上述问题，本文从地理信息注入方式出发，对原始方法进行分析与改进。本文首先比较了显式地理信息注入与隐式地理信息注入两类方案，发现将地理信息直接并入大语言模型输入或输出会显著增加序列建模复杂度，且对核心 SID 预测任务的帮助有限。因此，本文最终选择在 Semantic ID Construction 阶段引入隐式地理信息注入机制，利用文本编码器与 GeoPE（Geographical Positional Encoding）构造融合语义与空间信息的连续 POI 表示，再结合 RQ-VAE 学习地理增强的语义 ID，从而提升 POI 表示的空间感知能力，并为后续生成式推荐提供更高质量的离散表示。

\section{GNPR-SID 原始框架}
\label{sec:gnprsid_framework}

GNPR-SID 是一种面向生成式 Next POI Recommendation 任务的方法，其整体框架由两个核心模块组成，即 Semantic ID Construction 模块与 Generative POI Recommendation 模块。前者负责为每个 POI 学习具有层次结构和语义相似性的离散表示，后者则基于这些离散语义标识，对用户下一步可能访问的兴趣点进行生成式预测。其核心思想是利用语义化、结构化的离散标识替代传统推荐任务中无语义的随机 POI ID，从而使大语言模型能够在统一 token 空间中学习用户历史行为与目标兴趣点之间的映射关系。

\subsection{Semantic ID Construction 模块}

在 Semantic ID Construction 模块中，GNPR-SID 首先基于 POI 的多源属性构造连续表示。原始方法主要整合四类特征：类别信息、位置相关信息、时间信息以及用户协同信息。其中，类别信息用于描述 POI 的语义属性；位置相关信息由经纬度转换得到区域级地理编码，用于提供粗粒度的空间先验；时间信息由历史访问时间映射至离散时间槽，用于刻画 POI 的时间活跃模式；用户协同信息则利用高频访问用户及共访问关系来增强 POI 的行为关联表示。设上述四类特征分别记为 $c$、$r$、$t$ 和 $c_u$，则原始 GNPR-SID 中 POI 的连续表示可写为
\begin{equation}
\label{eq:gnprsid_pe}
p_e = \mathrm{concat}(c, r, t, c_u).
\end{equation}

在获得连续表示 $p_e$ 后，GNPR-SID 利用 RQ-VAE（Residual Quantized Variational AutoEncoder）对其进行多级残差量化，将连续空间中的 POI 表示映射为离散语义标识。设模型包含 $L$ 层量化码本，则每个 POI 在各层码本中分别对应一个 codeword index，最终其 Semantic ID 可表示为
\begin{equation}
SID(p) = [s_1, s_2, \dots, s_L],
\end{equation}
其中 $s_l$ 表示第 $l$ 层量化码本中所选中的索引。由于残差量化具有逐层细化表示的特点，语义相近的 POI 倾向于共享更长的 SID 前缀，因此该离散表示天然具有一定的层次结构与相似性表达能力。

为了学习高质量的离散语义表示，GNPR-SID 采用联合目标函数对 RQ-VAE 进行训练，其总体损失可表示为
\begin{equation}
\label{eq:gnprsid_rq_loss}
\mathcal{L}_{total} = \mathcal{L}_{recon} + \mu \mathcal{L}_{quant} + \lambda \mathcal{L}_{div},
\end{equation}
其中，$\mathcal{L}_{recon}$ 表示重建损失，用于约束量化表示能够恢复原始输入特征；$\mathcal{L}_{quant}$ 表示量化损失，用于优化编码器输出与码本向量之间的匹配关系；$\mathcal{L}_{div}$ 表示多样性损失，用于提升码本使用的均衡性，避免大量样本聚集到少量 codeword 上。

\subsection{Generative POI Recommendation 模块}

在 Generative POI Recommendation 模块中，GNPR-SID 将用户历史访问序列转化为结构化文本提示，并通过监督微调方式训练大语言模型完成下一兴趣点预测。设用户 $u$ 的历史访问序列为
\begin{equation}
H_u = \{(p_1,t_1),(p_2,t_2),\dots,(p_n,t_n)\},
\end{equation}
其中 $p_i$ 表示第 $i$ 次访问的兴趣点，$t_i$ 表示对应的访问时间。GNPR-SID 首先将每个 POI 替换为其对应的 SID token 序列，并与时间信息共同组织成输入提示。其典型输入形式可表示为：
\begin{quote}
\small
\texttt{user\_<uid> visited: <SID> at [time], ..., visited <SID> at [time]. When [time] user\_<uid> is likely to visit:}
\end{quote}

模型的输出目标则为下一兴趣点对应的 SID 序列。通过这种方式，GNPR-SID 将传统的 Next POI Recommendation 问题重构为一个条件生成问题，使大语言模型能够在离散语义空间中直接建模用户行为模式与兴趣点转移关系。

总体来看，GNPR-SID 的优势在于利用层次化离散语义表示替代无语义的随机 POI ID，从而增强生成式模型对 POI 语义相似性及行为模式的建模能力。然而，该方法在空间信息建模方面仍存在不足。虽然原始方法引入了区域级地理特征，但其地理建模方式仍偏粗粒度，且尚未与连续语义表示形成更深层次的耦合，这为本文的进一步改进提供了空间。

\section{问题分析与方法设计动机}
\label{sec:problem_analysis}

POI 推荐任务与一般序列推荐任务的重要区别在于，POI 本身具有显著的空间属性。用户下一步访问的兴趣点不仅受到历史兴趣偏好的影响，还与当前位置、空间可达性、活动半径以及城市功能区域分布密切相关。因此，如果离散语义标识仅由类别、时间和协同信息等特征学习得到，而没有充分建模真实地理空间关系，那么最终得到的 SID 对 POI 的表达能力仍然是不完整的。

在 GNPR-SID 原始框架中，空间信息主要通过区域级地理编码以特征拼接的方式加入连续表示。这种做法虽然能够提供一定的区域先验，但仍存在以下两个问题。

第一，区域编码本质上更接近符号化标签，只能反映粗粒度的空间位置，难以刻画 POI 之间更加细粒度的地理差异。例如，同属某一区域内的两个 POI 可能在真实距离、方向关系和功能邻近性上存在明显差别，但这些差别在单一的区域编码中难以体现。

第二，地理信息与语义信息之间主要表现为简单拼接关系，而不是在表示学习过程中形成统一的潜在空间结构。换言之，原始方法虽然“加入了地理信息”，但这种加入方式并未真正改变 POI 语义表示的内部几何结构，因此空间信息对最终 SID 学习结果的影响仍然有限。

基于上述分析，本文的核心思路是：不再将地理信息视为一个附加特征，而是将其视为一种能够作用于语义表示的变换机制，使空间关系在表示学习阶段直接影响 POI 的连续表示构造。围绕这一目标，本文先后探索了显式地理信息注入与隐式地理信息注入两类方案，并最终选择在 Semantic ID Construction 阶段引入隐式空间增强表示。

图 \ref{fig:baseline_vs_ours} 从流程层面对比了 Baseline 与本文在 Semantic ID 构造阶段的差异：二者在数据预处理及下游生成式推荐上保持一致，主要分歧出现在进入 RQ-VAE 之前的 POI 连续表征构造路径。

\section{显式地理信息注入方案及其局限性}
\label{sec:explicit_geo_attempt}

在方法探索初期，本文首先尝试通过显式方式将地理信息注入 Generative POI Recommendation 模块。该方案的基本出发点在于：既然 GNPR-SID 最终通过大语言模型生成下一兴趣点的 SID，那么可以进一步将地理位置编码组织为显式 token，并直接注入到历史访问序列提示词中，甚至将其设计为输出目标的一部分，从而使模型在生成下一 POI 时能够显式感知空间信息。

具体而言，本文考虑采用 Geohash 等离散地理编码方式，将经纬度映射为统一的地理 token 序列，并与原始 SID 共同构成输入或输出表示。在这种设计下，历史访问序列中的每个 POI 不再仅由其语义 SID 表示，而是扩展为“语义 token + 地理 token”的联合表示。相应地，模型输出也可设计为目标 POI 的地理 token 与 SID token 的联合结果。

从直觉上看，该方案能够使地理信息在 token 层面被大语言模型直接“看见”，似乎有助于模型学习 POI 之间的区域相似性与空间迁移模式。然而，实验观察表明，该方案存在明显局限。
首先，显式地理 token 的引入会直接增加输入与输出序列长度。对于自回归生成模型而言，序列长度的增加意味着训练成本和推理成本同步上升，尤其是在历史访问序列本身已经较长的情况下，这种额外开销更加明显。
其次，当地理 token 与 SID token 并列出现在输出端时，模型事实上需要同时完成地理预测与语义预测两个子任务，导致训练目标被分散。当地理 token 与核心 SID 判别之间不能形成稳定、直接的帮助关系时，这种联合建模反而可能削弱模型对核心推荐目标的关注，从而影响下一 POI 预测精度。
最后，显式地理信息虽然增加了模型对空间信息的可见性，但并不意味着这些信息能够有效转化为更好的 SID 预测能力。实验中可以观察到，显式地理 token 往往会提高生成复杂度，却未必带来稳定的性能收益，甚至可能由于输出空间膨胀而导致模型效果下降。

综合上述分析，本文认为显式地理信息注入方案虽然在形式上更直接，但会显著增加生成式建模复杂度，且不利于将地理信息稳定地转化为对核心推荐任务的有效帮助。因此，本文最终没有采用该方案，而是转向在表示学习阶段完成地理信息隐式注入的思路。

\section{基于 GeoPE 的隐式地理增强表示构建}
\label{sec:implicit_geope}

与显式注入方案不同，隐式地理信息注入更关注在连续表示层面建模空间关系，并通过连续表示的改变影响最终 SID 的学习结果。本文借鉴基于旋转位置编码的空间建模思想，在 POI 表示学习阶段引入 GeoPE（Geographical Positional Encoding）机制，使地理信息能够以结构化方式嵌入潜在表示空间，从而得到同时包含语义信息与空间信息的连续 POI 表示。

从 Semantic ID Construction 的目标来看，本文期望最终学习得到的 SID 至少具备以下三方面性质。第一，\textbf{语义表达性}，即离散标识应尽可能保留 POI 的类别语义信息，使同类或功能相近的 POI 在表示空间中具有可辨识的相似性。第二，\textbf{空间敏感性}，即 SID 不仅反映 POI 的语义属性，还能够区分其在真实地理空间中的相对位置差异，使地理邻近性与空间结构能够通过离散标识间接体现出来。第三，\textbf{层次可区分性}，即语义相近或空间关系相近的 POI 应倾向于共享更长的 SID 前缀，而差异较大的 POI 则在更高层级上被区分开来。围绕这三个目标，本文先构造语义--空间联合连续表示，再通过 RQ-VAE 将其映射到具有层次结构的离散语义 ID 空间。

\subsection{基础语义表示构建}

设某一 POI 的文本属性记为 $A_p$。在本文所使用的数据集中，POI 缺少丰富的自然语言描述信息，能够直接利用的文本属性主要为类别名称。因此，本文首先采用轻量级文本编码器对类别属性进行编码，得到基础 POI 语义表示：
\begin{equation}
x_p = f_{\mathrm{text}}(A_p),
\end{equation}
其中，$x_p \in \mathbb{R}^{D}$ 表示 POI 的基础语义嵌入，$f_{\mathrm{text}}(\cdot)$ 表示文本编码器。本文具体采用预训练模型 \texttt{all-MiniLM-L6-v2} 作为文本编码器，其输出维度为 $D=384$。之所以选择该模型，一方面是因为类别名称文本较短，使用轻量模型即可获得较稳定的语义表示；另一方面，较小的编码器规模也有助于控制整体训练成本。

\subsection{参考点集合构建}

为了建模空间依赖关系，本文不直接对经纬度数值进行拼接，而是首先基于所有 POI 的空间分布构造若干参考点。具体地，设所有 POI 的经纬度坐标集合为 $\mathcal{L}=\{l_p\}_{p=1}^{N}$，其中 $l_p=(lat_p, lon_p)$。本文采用 K-Means 聚类方法对所有 POI 的坐标进行聚类，得到 $\Omega$ 个参考点：
\begin{equation}
R = \{R_1, R_2, \dots, R_{\Omega}\}.
\end{equation}
其中，每个参考点 $R_{\omega}$ 可视为一个空间密度中心，用于刻画城市空间中的典型区域结构。通过引入多个参考点，本文可以从相对位置而非绝对坐标的角度对 POI 的空间关系进行建模。

\subsection{方位角计算}

对于任意 POI $p$ 及参考点 $R_{\omega}$，本文计算两者之间的相对方位角 $\theta_{p,\omega}$，用于表示 POI 相对于该参考点的方向关系。记参考点坐标为 $R_{\omega}=(lat_{\omega}, lon_{\omega})$，则可通过如下形式计算对应的方位角：
\begin{equation}
\theta_{p,\omega} = \mathrm{atan2}(lat_p-lat_{\omega},\, lon_p-lon_{\omega}).
\end{equation}

对于每一个 POI，关于全部 $\Omega$ 个参考点可以得到一组方位角
\begin{equation}
\Theta_p = \{\theta_{p,1}, \theta_{p,2}, \dots, \theta_{p,\Omega}\}.
\end{equation}
该角度集合从几何上刻画了 POI 相对于多个空间锚点的方向关系，因而可以视为对其真实位置的一种隐式表示。与直接使用绝对经纬度不同，这种相对位置建模方式更强调 POI 在整体空间结构中的关系信息。

\subsection{基于旋转编码的空间信息注入}

在获得基础语义表示 $x_p$ 与对应的方位角集合 $\Theta_p$ 后，本文采用旋转变换的方式将地理信息注入语义向量。具体而言，首先将基础嵌入 $x_p$ 划分为 $\Omega$ 个子向量块：
\begin{equation}
x_p = [x_p^{(1)}; x_p^{(2)}; \dots; x_p^{(\Omega)}],
\end{equation}
其中，$x_p^{(\omega)}$ 表示第 $\omega$ 个子向量块，对应参考点 $R_{\omega}$。为便于进行二维旋转操作，要求每个子块的维度为偶数；当 $D$ 不能被 $\Omega$ 整除时，可通过线性投影或零填充方式将其映射到合适维度。

随后，对于第 $\omega$ 个子向量块中的相邻维度对，施加由方位角 $\theta_{p,\omega}$ 控制的二维旋转。设该子块中的某一对相邻维度为 $(x_{2i}^{(\omega)}, x_{2i+1}^{(\omega)})$，则其旋转结果表示为
\begin{equation}
\begin{bmatrix}
\hat{x}_{2i}^{(\omega)} \\
\hat{x}_{2i+1}^{(\omega)}
\end{bmatrix}
=
\begin{bmatrix}
\cos \theta_{p,\omega} & -\sin \theta_{p,\omega} \\
\sin \theta_{p,\omega} & \cos \theta_{p,\omega}
\end{bmatrix}
\begin{bmatrix}
x_{2i}^{(\omega)} \\
x_{2i+1}^{(\omega)}
\end{bmatrix}.
\end{equation}

对全部子向量块完成旋转操作后，将其重新拼接，得到最终的地理增强连续表示：
\begin{equation}
x_p' = \mathrm{GeoPE}(x_p, l_p).
\end{equation}

通过上述构造，原本仅包含类别语义信息的基础表示 $x_p$ 被映射为同时融合语义信息与空间信息的增强表示 $x_p'$。与直接拼接经纬度或区域编码的方式相比，GeoPE 并不将地理信息作为外部附加特征，而是通过对语义向量内部结构施加几何变换，使空间差异成为潜在表示的一部分。这意味着，即使两个 POI 属于相同类别，只要它们处于不同地理位置，其最终连续表示仍会因相对参考点角度不同而产生差异，从而提升表示的空间判别能力。

\section{地理增强 SID 的学习及其在生成式推荐中的接入}
\label{sec:geo_sid_learning}

在获得地理增强表示 $x_p'$ 后，本文以之取代第 \ref{sec:gnprsid_framework} 节中式 \eqref{eq:gnprsid_pe} 所示的多源显式拼接向量 $p_e$，作为 RQ-VAE 的量化输入；量化结构、层次化码本与残差量化过程与 GNPR-SID 相同，故不再赘述，仅给出与本文记号直接衔接的核心关系。设编码器输出为
\begin{equation}
z_p = \mathrm{Enc}(x_p'),
\end{equation}
第 $l$ 层码本为 $\mathcal{B}^{(l)}$，残差量化满足 $r_p^{(0)}=z_p$，
\begin{equation}
q_p^{(l)}=\arg\min_{e \in \mathcal{B}^{(l)}} \| r_p^{(l-1)}-e \|_2^2,\quad
r_p^{(l)} = r_p^{(l-1)} - q_p^{(l)},\quad l=1,\dots,L,
\end{equation}
量化向量求和重建 $\hat{z}_p = \sum_{l=1}^{L} q_p^{(l)}$，解码器满足 $\hat{x}_p' = \mathrm{Dec}(\hat{z}_p)$。每层所选 codeword 的索引构成离散语义 ID，形式与第 \ref{sec:gnprsid_framework} 节一致：
\begin{equation}
SID(p) = [s_1, s_2, \dots, s_L],\quad s_l=\mathrm{index}(q_p^{(l)}).
\end{equation}
由于 $x_p'$ 已融合 GeoPE 注入的空间信息，上述 SID 在保持「层次前缀相似性」这一 GNPR-SID 性质的同时，对 POI 的地理差异更为敏感。RQ-VAE 训练仍采用式 \eqref{eq:gnprsid_rq_loss} 形式的重建、量化与多样性联合损失；地理信息不显式作为独立 token 输出，而由连续表示与量化过程隐式写入 SID。

在生成式推荐阶段，本文完全沿用第 \ref{sec:gnprsid_framework} 节中的提示构造与监督微调设定：历史序列 $H_u$、提示模板及「下一 SID」生成目标均与 Baseline 一致，区别仅在于各 POI 的 \texttt{<SID>} 来自基于 $x_p'$ 的 RQ-VAE，而非原始多源特征拼接。将 POI 映射为 SID 的函数仍记为 $\mathcal{F}:p\mapsto SID(p)$，多级索引可展开为各层 token 序列，例如
\begin{equation}
T(p)=\langle a_{s_1}\rangle \langle b_{s_2}\rangle \cdots \langle l_{s_L}\rangle.
\end{equation}
据此得到带时间戳的语义 ID 序列后，即可按第 \ref{sec:gnprsid_framework} 节所述方式组织训练样本；推理时仍由预测 SID 查表映射回 POI。本文未改动 GNPR-SID 生成端词表与输出形式，改进集中于 Semantic ID Construction 前端，使空间信息在离散表示学习阶段进入 SID，从而在无需额外地理 token 的前提下提升下一兴趣点预测可用的表示质量。

\section{本章小结}
\label{sec:method_summary}

本章围绕本文提出的基于隐式地理信息注入的 POI 语义表示构建方法进行了系统介绍。首先，梳理了 GNPR-SID 的原始框架，说明其由 Semantic ID Construction 模块与 Generative POI Recommendation 模块组成，并分析了其在地理信息建模方面存在的不足。随后，本文讨论了显式地理信息注入方案及其局限性，指出将地理信息直接作为额外 token 注入生成模型虽然形式直观，但会显著增加序列长度和输出复杂度，且难以稳定提升核心推荐性能。

在此基础上，本文提出了一种基于 GeoPE 的隐式地理信息注入方法。该方法首先利用轻量文本编码器对 POI 类别属性进行编码，得到基础语义表示；随后，基于多个参考点与 POI 之间的相对方位角，对语义向量进行分块旋转，从而构造同时融合语义信息与空间信息的连续表示。在此基础上，本文进一步借助 RQ-VAE 对地理增强表示进行多层残差量化，使 Semantic ID 的学习过程能够直接作用于语义--空间联合潜在空间，进而得到兼具语义表达能力、空间敏感能力与层次结构特征的地理增强 SID。

最后，本文说明了如何将地理增强 SID 无缝接入 GNPR-SID 的生成式推荐框架。与显式地理 token 注入方案不同，本文并不改变大语言模型的输出形式，而是通过优化前端离散表示学习过程，使生成模型在维持原有建模范式与输出复杂度不变的条件下，获得更具空间判别能力的目标表示，为后续实验中的性能提升奠定了方法基础。
