\chapter{基于解耦双轨制ID的生成式推荐模型}
\label{chap:method}

\section{模型总体架构}
针对现有混合量化策略导致的码本碰撞问题，本文提出了一种基于解耦双轨制 ID 的生成式 POI 推荐模型。该模型主要由两部分构成：基于解耦策略的时空双轨制 ID 构造模块，以及基于 LLaMA-3 的生成式推荐模块。

\section{刚性物理空间前缀编码}
在生成式框架下，大模型对实体空间位置的感知完全依赖于输入的 Semantic ID。为彻底根除物理空间的坍塌错乱，本文摒弃了利用神经网络模糊压缩位置信息的方案。对于给定的 POI 经纬度坐标 $(lat, lon)$，直接调用 Geohash 算法生成固定精度的空间散列编码。
为便于大模型学习空间的层级递进关系，本文将生成的 Geohash 字符串拆分为多个 Token 作为 ID 的前缀序列。例如，将长度为 6 的散列值 \texttt{dr5rev} 拆分为 $\langle geo\_dr5 \rangle \langle geo\_rev \rangle$。这一前缀构成了 POI 绝对物理位置的刚性约束。

\section{柔性协同语义后缀编码}
在剥离了地理位置特征后，本文为 RQ-VAE 模型进行了有效“减负”。重构后的 POI 输入特征向量 $\mathbf{p}_{sem}$ 仅由类别分布（Category）、时间属性（Time slots）以及最频繁访客的共现信号（Co-visitation patterns）拼接而成。
将其输入浅层 RQ-VAE 模型进行特征压缩，最终生成形如 $\langle sem\_14 \rangle \langle sem\_05 \rangle$ 的语义后缀。在该机制下，若不同 POI 发生了码本碰撞，则代表它们在“用户偏好与业务属性”上高度同构（例如同为早晨热门的咖啡厅），由于它们已被不同的刚性地理前缀区分，此类语义碰撞不仅无害，反而有助于 LLM 进行泛化推荐。

\section{混合ID序列组织与微调指令构建}
结合空间前缀与语义后缀，一个 POI 的完整双轨制 ID 序列被定义为：
\begin{equation}
    \text{ID}_{POI} = [\langle geo\_prefix \rangle, \langle geo\_suffix \rangle, \langle sem\_layer1 \rangle, \langle sem\_layer2 \rangle]
\end{equation}

在数据构建阶段，采用滑动窗口（Sliding Window）方法，严格按照时间先后顺序截取用户的历史访问记录以防止数据泄露（Data Leakage）。将用户的历史轨迹序列转化为自然语言指令（Prompt），格式如下：
\begin{quotation}
    User visited: $\langle geo\_dr7 \rangle \langle geo\_2hd \rangle \langle sem\_14 \rangle \langle sem\_05 \rangle$ at 2012-07-08 13:47, ... When 2012-11-06 17:07 user is likely to visit:
\end{quotation}

\section{基于 LLaMA-3-8B 的两段式生成推理}
将构建好的时序指令集输入 LLaMA-3-8B 进行 LoRA 监督微调。在推理阶段，由于双轨制 ID 的巧妙设计，大语言模型展现出了符合人类决策直觉的“两段式”推理逻辑：首先根据历史轨迹的时空转移规律，自回归地生成目标商圈的地理网格 Token；随后，基于用户的长程兴趣偏好，在预测的网格内生成具体的语义属性 Token，最终精准命中下一个目标 POI。
